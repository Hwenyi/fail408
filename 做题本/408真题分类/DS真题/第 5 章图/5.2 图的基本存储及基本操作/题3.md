## 题3
### 题目
【2021】已知无向连通图 $G$ 中顶点集 $V$ 和边集 $E$ 组成, $| E|  > 0$ ,当 $G$ 中度为奇数的顶点个数为不大于 2 的偶数时, $G$ 存在包含所有边且长度为 $| E|$ 的路径 (称为 EL路径)。设图 $G$ 采用邻接矩阵存储,类型定义如下:

```cpp
typedef struct {
    // 图的定义
    int numVertices, numEdges; // 图中实际的顶点数和边数
    char VerticesList[MAXV]; // 顶点表。MAXV 为已定义常量
    int Edge[MAXV][MAXV]; // 邻接矩阵
} MGraph;
```

请设计算法: `int IsExistEL (MGraph G)`，判断 G 是否存在 EL 路径，若存在，则返回 1 ，否则返回 0。要求:

1. 给出算法的基本设计思想。
2. 根据设计思想，采用 C 或 C++ 语言描述算法，关键之处给出注释。
3. 说明你所设计算法的时间复杂度和空间复杂度。
### 分析
如何理解这句话：当 $G$ 中度为奇数的顶点个数为不大于 2 的偶数时。也就是奇数度的点的个数为 0 或者 2 个。

> [!NOTE] 下面统计顶点的度是多余的
> 根据题意，这里的奇数度的个数已经是充要的了，只需要统计数量就行了

我们分情况讨论：
1. 当这个奇数度的点的个数为 0 时，所有的点都是偶数度的，因为这是无向图，所以所有点的度之和就是边数的两倍，也就是 $2k_{1}V_{1}+2k_{2}V_{2}+\dots {2}k_{n}V_{n}=2E$，我们只需要遍历这个图统计每个点的度，如果度之和等于边数就存在 EL 路径，还得设置一个计数器，保证奇数度的点的数量确实是 0 个。
2. 当奇数度的点的个数为 2 时，两个奇数之和仍是偶数，所有点的度之和再 +1 应该是边数的两倍，计数器保证奇数点的数量确实是 2
现在我们回忆一下如何遍历[[邻接矩阵]]，这里的边数我们是知道的，顶点总数也知道，只需要在遍历的时候把度统计出来

[[时间复杂度]]上因为需要遍历整个图，所以是$O(n^{2})$的，[[空间复杂度]]上，因为需要用到一个辅助数组来存每个节点的度，所以是$O(n)$
